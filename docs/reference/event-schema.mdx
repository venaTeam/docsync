docsync's CI entry point is **event-native**: it derives every diff parameter automatically from the CI environment instead of requiring the caller to pass `--src-repo/--base/--head/--pr-number/--pr-title` flags. This serves the project's core goal ‚Äî *triggered by commits, not many parameters*.

The parameters come from the GitHub event JSON GitHub writes to `$GITHUB_EVENT_PATH`, or from GitLab's predefined `CI_*` variables (auto-detected via the `GITLAB_CI` env var).

This page documents the event ingestion module (`docsync.events`) and the data contracts it produces (`docsync.models`): the event shapes understood, the functions that parse them, and the schema of every model exchanged across the pipeline.

## Event ingestion pipeline

<Steps>
  <Step title="Auto-detect the platform">
    `diff_from_ci` checks `GITLAB_CI` (GitLab env vars) first, then `$GITHUB_EVENT_PATH` (the GitHub event file).
  </Step>
  <Step title="Read the event file">
    `load_event($GITHUB_EVENT_PATH)` reads and JSON-parses the triggering event GitHub wrote to disk.
  </Step>
  <Step title="Parse the parameters">
    `parse_event(event)` extracts the comparison parameters into an `EventInfo` (pure, no I/O).
  </Step>
  <Step title="Apply the null-base fallback">
    If the recovered base is git's all-zeros null SHA, it is rewritten to `<head>^`.
  </Step>
  <Step title="Delegate to the differ">
    `diff_from_event` calls `diff_github(repo, base, head, pr_number=‚Ä¶, pr_title=‚Ä¶)`, returning a `CodeDiff`.
  </Step>
</Steps>

## Supported event shapes

`parse_event` recognizes two shapes, tried in this order.

### 1. `repository_dispatch`

A hand-rolled trigger carrying an explicit `client_payload`. Selected whenever `event["client_payload"]` is present.

| `client_payload` key | Maps to | Required | Notes |
|----------------------|---------|----------|-------|
| `repo` | `EventInfo.repo` | Yes | `owner/name` or a local path |
| `base_sha` | `EventInfo.base_sha` | Yes | base of the comparison |
| `head_sha` | `EventInfo.head_sha` | Yes | head of the comparison |
| `pr_number` | `EventInfo.pr_number` | No | coerced to `int`; `None`/`""` ‚Üí `None` |
| `pr_title` | `EventInfo.pr_title` | No | `None` when absent |

```json
{
  "client_payload": {
    "repo": "keephq/keep-api-gateway",
    "base_sha": "a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2",
    "head_sha": "f6e5d4c3b2a1f6e5d4c3b2a1f6e5d4c3b2a1f6e5",
    "pr_number": "42",
    "pr_title": "Add dedup endpoint"
  }
}
```

<Note>
`pr_number` is coerced with best-effort int parsing (`_coerce_pr_number`), because JSON may carry it as a string (e.g. `"42"`). A value of `None` or the empty string becomes `None`.
</Note>

### 2. `push`

The native GitHub push event. Selected when the event has both `before` and `after` keys.

| Event field | Maps to | Notes |
|-------------|---------|-------|
| `repository.full_name` | `EventInfo.repo` | required (missing `repository` is treated as `{}`, raising `KeyError`) |
| `before` | `EventInfo.base_sha` | the pre-push tip; all-zeros on first push |
| `after` | `EventInfo.head_sha` | the new tip |
| `head_commit.message` | `EventInfo.pr_title` | `None` when `head_commit` is absent |
| ‚Äî | `EventInfo.pr_number` | always `None` for push events |

```json
{
  "before": "0000000000000000000000000000000000000000",
  "after": "f6e5d4c3b2a1f6e5d4c3b2a1f6e5d4c3b2a1f6e5",
  "repository": { "full_name": "keephq/keep-api-gateway" },
  "head_commit": { "message": "Refactor alert deduplicator" }
}
```

<Warning>
If the event matches neither shape ‚Äî no `client_payload` and no `before`/`after` keys ‚Äî `parse_event` raises `ValueError`.
</Warning>

### GitLab CI (environment variables)

GitLab CI does not write an event JSON file; it exposes [predefined variables](https://docs.gitlab.com/ci/variables/predefined_variables/) instead. `parse_gitlab_env(env)` recovers the same `EventInfo` from them. Merge-request pipelines (`CI_PIPELINE_SOURCE=merge_request_event`) expose the richer `CI_MERGE_REQUEST_*` family, preferred when present.

| Env var | Maps to | Notes |
|---------|---------|-------|
| `CI_PROJECT_PATH` | `EventInfo.repo` | `group/name`, matching GitHub's `owner/name` shape |
| `CI_COMMIT_BEFORE_SHA` | `EventInfo.base_sha` | all-zeros on a branch's first push (null-base fallback) |
| `CI_COMMIT_SHA` | `EventInfo.head_sha` | the commit tip (ephemeral merge-result commit on MR pipelines) |
| `CI_MERGE_REQUEST_IID` | `EventInfo.pr_number` | MR pipelines; coerced to `int` |
| `CI_MERGE_REQUEST_TITLE` | `EventInfo.pr_title` | MR pipelines |
| `CI_MERGE_REQUEST_DIFF_BASE_SHA` | `EventInfo.base_sha` | MR merge-base; preferred over `CI_COMMIT_BEFORE_SHA` |
| `CI_MERGE_REQUEST_SOURCE_BRANCH_SHA` | `EventInfo.head_sha` | real source-tip; preferred over `CI_COMMIT_SHA` |

Raises `ValueError` if neither `CI_PROJECT_PATH` nor the commit SHAs are set.

## The null SHA

GitHub uses an all-zeros SHA as the "base" of a branch's very first push (there is no prior commit to compare against). It also appears as the base of a `repository_dispatch` payload with no real ancestor.

| Symbol | Value / signature | Description |
|--------|-------------------|-------------|
| `NULL_SHA` | `"0" * 40` | The canonical 40-character all-zeros SHA. |
| `is_null_sha(sha)` | `(str \| None) -> bool` | `True` when `sha` is truthy and consists only of zeros (any length). Returns `False` for `None` or `""`. |

When `diff_from_event` detects a null base via `is_null_sha`, it rewrites the base to `f"{info.head_sha}^"` ‚Äî the head commit's first parent ‚Äî so the diff is still meaningful instead of being taken against the empty tree.

## Function reference

### `parse_event(event: dict) -> EventInfo`

Extract diff parameters from a GitHub event payload. Pure ‚Äî performs no I/O. Recognizes `repository_dispatch` then `push` (see [Supported event shapes](#supported-event-shapes)). Raises `ValueError` if neither shape matches.

### `parse_gitlab_env(env: Mapping[str, str]) -> EventInfo`

Extract diff parameters from GitLab CI environment variables (pure; no I/O). See [GitLab CI (environment variables)](#gitlab-ci-environment-variables) for the variable mapping. Prefers the `CI_MERGE_REQUEST_*` family on merge-request pipelines. Raises `ValueError` if neither `CI_PROJECT_PATH` nor the commit SHAs are set.

### `load_event(event_path: str | Path) -> dict`

Read and JSON-parse the event file at `event_path` (i.e. `$GITHUB_EVENT_PATH`). Returns the raw event `dict`.

### `diff_from_event(event_path, *, runner=None) -> CodeDiff`

Build a `CodeDiff` from a GitHub event file end to end: reads the JSON at `event_path`, parses it with `parse_event`, applies the null-base fallback, and delegates to the diffing callable.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `event_path` | `str \| Path` | ‚Äî | Path to the event JSON file (`$GITHUB_EVENT_PATH` in CI). |
| `runner` | `Optional[Callable[..., CodeDiff]]` | `None` | The diffing callable. When `None`, the module-level `diff_github` is resolved **at call time** ‚Äî so tests can pass a network-free stub or monkeypatch `docsync.events.diff_github`. |

The runner is invoked as:

```python
diff_fn(
    info.repo,
    base,                 # info.base_sha, or f"{info.head_sha}^" on a null base
    info.head_sha,
    pr_number=info.pr_number,
    pr_title=info.pr_title,
)
```

    from docsync.events import diff_from_event

    # GitHub sets $GITHUB_EVENT_PATH to the triggering event JSON.
    import os
    code_diff = diff_from_event(os.environ["GITHUB_EVENT_PATH"])
    print(code_diff.changed_paths())
    ```

    diff = diff_from_event("event.json", runner=fake_runner)
    ```
  </Tab>

### `diff_from_gitlab_env(env=None, *, runner=None) -> CodeDiff`

The GitLab analogue of `diff_from_event`: there is no event file, so the parameters come from the predefined `CI_*` variables via `parse_gitlab_env`. `env` defaults to `os.environ` when `None`; the same null-base fallback applies (GitLab sets `CI_COMMIT_BEFORE_SHA` to all-zeros on a branch's first push). `runner` mirrors `diff_from_event`.

### `diff_from_ci(env=None, *, runner=None) -> CodeDiff`

Auto-detect the CI platform and build a `CodeDiff`. Detection order: `GITLAB_CI` set ‚Üí `diff_from_gitlab_env`; else `GITHUB_EVENT_PATH` set ‚Üí `diff_from_event`. `env` defaults to `os.environ`. Raises `ValueError` if neither platform is detected. GitHub behavior is unchanged ‚Äî when `GITLAB_CI` is absent this falls through to the existing GitHub event-file path.

## `EventInfo`

The diff parameters recovered from a CI event (GitHub or GitLab), **before** the null-base fixup. A `@dataclass`.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `repo` | `str` | ‚Äî | Repository identifier (`owner/name` or local path). |
| `base_sha` | `str` | ‚Äî | Base of the comparison (may be a null SHA). |
| `head_sha` | `str` | ‚Äî | Head of the comparison. |
| `pr_number` | `Optional[int]` | `None` | PR number, when known. |
| `pr_title` | `Optional[str]` | `None` | PR title or commit message, when available. |

<Note>
`EventInfo` carries the base **as recovered** ‚Äî the null-base fixup (`<head>^`) is applied by `diff_from_event`, not by `parse_event`. If you call `parse_event` directly you receive the raw base SHA.
</Note>

## Data models

The models below are defined in `docsync/models.py`, the single seam every pipeline stage builds against. They are **Pydantic v2** models (this is a standalone tool, unrelated to Keep's Pydantic v1 services), chosen because the Anthropic SDK's `messages.parse()` structured-output helper validates against Pydantic models directly.

### `FileStatus` (enum)

A string enum describing how a file changed in a diff.

| Member | Value |
|--------|-------|
| `FileStatus.ADDED` | `"added"` |
| `FileStatus.MODIFIED` | `"modified"` |
| `FileStatus.REMOVED` | `"removed"` |
| `FileStatus.RENAMED` | `"renamed"` |

### `ChangedFile`

One file touched by a diff, with its hunks and the code symbols affected.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `path` | `str` | ‚Äî | The file's path. |
| `status` | `FileStatus` | ‚Äî | How the file changed. |
| `previous_path` | `Optional[str]` | `None` | Set when `status == RENAMED`. |
| `hunks` | `list[str]` | `[]` | Unified-diff hunk texts (the `@@ ... @@` blocks + context), one entry per hunk. |
| `changed_symbols` | `list[str]` | `[]` | Function / class / module-level assignment names whose body the hunks touch. The cross-boundary signal used by impact mapping; survives line-number churn. |

### `CodeDiff`

The structured result of comparing `base..head` in a single service repo ‚Äî the value `diff_from_event` ultimately returns.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `repo` | `str` | ‚Äî | e.g. `"keephq/keep-api-gateway"` (`owner/name`) or a local path. |
| `base_sha` | `str` | ‚Äî | Base commit SHA. |
| `head_sha` | `str` | ‚Äî | Head commit SHA. |
| `pr_number` | `Optional[int]` | `None` | PR number, when known. |
| `pr_title` | `Optional[str]` | `None` | PR title, when known. |
| `files` | `list[ChangedFile]` | `[]` | The changed files. |

**Methods**

| Method | Returns | Description |
|--------|---------|-------------|
| `changed_paths()` | `list[str]` | Every file `path`, plus each `previous_path` for renamed files. |
| `all_symbols()` | `list[str]` | De-duplicated union of `changed_symbols` across all files, preserving first-seen order. |

```python
diff = CodeDiff(
    repo="keephq/keep-api-gateway",
    base_sha="a1b2c3d4...",
    head_sha="f6e5d4c3...",
    pr_number=42,
    pr_title="Add dedup endpoint",
    files=[
        ChangedFile(
            path="src/routes/alerts.py",
            status=FileStatus.MODIFIED,
            hunks=["@@ -10,6 +10,8 @@ def list_alerts(): ..."],
            changed_symbols=["list_alerts", "AlertDto"],
        ),
    ],
)

diff.changed_paths()  # ["src/routes/alerts.py"]
diff.all_symbols()    # ["list_alerts", "AlertDto"]
```

### `SourceUnit`

One documentable source file, distilled to what the bootstrap planner needs. Lightweight by design ‚Äî paths and symbol names only, no file bodies ‚Äî because a whole service repo's worth of these has to fit in the planner's context; excerpts are fetched per-page only at author time.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `path` | `str` | ‚Äî | Repo-relative path, e.g. `"src/routes/alerts.py"`. |
| `kind` | `str` | ‚Äî | Coarse language/role tag: `"python"`, `"typescript"`, or `"other"`. |
| `symbols` | `list[str]` | `[]` | Top-level defs / classes / exports. |

`SourceUnit`s are aggregated into a `RepoDigest` (`repo`, `root`, `units`), whose `all_symbols()` returns the de-duplicated union of every unit's `symbols`, mirroring `CodeDiff.all_symbols()`.

## Relationship to the rest of the pipeline

The `repo` field is the join key across stages: a `CodeDiff.repo` (from an event), a `RepoDigest.repo` (from bootstrap ingest), and the `repo` on a page's anchors (`PlannedSource.repo` / `ManifestSource.repo`) all use the same `owner/name`-or-local-path identifier, so changed paths and symbols recovered from an event can be matched back to the documentation pages they affect.

<Note>
`diff_from_event` is intentionally network-free at the parsing layer. All network access lives behind `diff_github`, which is resolved at call time ‚Äî keep that boundary intact when extending event support so the module stays unit-testable.
</Note>
